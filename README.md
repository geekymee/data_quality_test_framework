# DE Test Framework

A two-step data quality framework for AWS Athena. Define your tests interactively in the terminal, then run them ‚Äî violations are uploaded to S3 as CSVs and a summary is printed at the end.

## Setup

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

### 2. Configure AWS credentials

Any one of:

```bash
# Option A ‚Äî environment variables
export AWS_ACCESS_KEY_ID=...
export AWS_SECRET_ACCESS_KEY=...
export AWS_DEFAULT_REGION=ap-south-1   # overrides hardcoded default in run_tests.py

# Option B ‚Äî AWS CLI profile
aws configure

# Option C ‚Äî IAM role (EC2 / ECS / Lambda) ‚Äî no action needed
```

## Usage

### Step 1 ‚Äî Build config (run once, or whenever tests change)

```bash
python build_config.py
```

The CLI will prompt you for:

| Prompt | Example |
|--------|---------|
| Athena database name | `my_database` |
| Table name | `orders` |
| Table type | `ICEBERG` |
| Table S3 location | `s3://my-bucket/iceberg/orders/` |
| Output S3 bucket | `my-dq-results-bucket` |
| Output S3 prefix | `dq-results/` |
| Athena temp S3 path | `s3://my-dq-results-bucket/athena-temp/` |
| fail_fast | `n` |
| Max violations per test | `1000` |
| Log level | `INFO` |

Then choose which test types to configure:

```
[1] not_null_single       ‚Äî one column must not be null
[2] not_null_composite    ‚Äî any of N columns must not be null
[3] no_duplicates         ‚Äî combination of columns must be unique
[4] datatype_checks       ‚Äî column values must be castable to expected type
[5] required_rows         ‚Äî all rows from a reference CSV must exist in the table
```

For each selected type you'll be prompted to add individual checks one at a time.

The result is written to `config/config.py`. You can edit it directly for quick tweaks without re-running the builder.

### Step 2 ‚Äî Run tests

```bash
python run_tests.py
```

What happens:

1. `config/config.py` is read ‚Äî table name, S3 paths, and all test definitions.
2. `sql_generator.py` builds one Athena SQL query per test at runtime.
3. Each query is submitted to Athena and polled until complete.
4. **PASS** ‚Äî no rows returned ‚Üí logged and added to summary.
5. **FAIL** ‚Äî rows returned ‚Üí enriched with `_test_name` / `_run_at` / `_test_type` columns and uploaded to S3 as a CSV.
6. A `_summary.csv` is uploaded containing one row per test.
7. If any test FAILed or ERRORed, the non-passing rows are printed and the process exits with code `1` (useful for CI).

## S3 output layout

```
s3://<bucket>/<prefix>/
‚îî‚îÄ‚îÄ <run_id>/                         e.g. 20240601T120000Z/
    ‚îú‚îÄ‚îÄ _summary.csv                  ‚Üê one row per test (always written)
    ‚îú‚îÄ‚îÄ check_id_not_null.csv         ‚Üê violation rows (only if FAIL)
    ‚îú‚îÄ‚îÄ check_order_fields_not_null.csv
    ‚îú‚îÄ‚îÄ check_id_unique.csv
    ‚îî‚îÄ‚îÄ ...
```

### Summary CSV columns

| Column | Description |
|--------|-------------|
| `test_name` | Name of the test case |
| `test_type` | `not_null_single`, `not_null_composite`, `no_duplicates`, `datatype_mismatch`, `required_rows` |
| `status` | `PASS`, `FAIL`, or `ERROR` |
| `violation_count` | Number of bad rows found (0 for PASS) |
| `output_s3_uri` | S3 URI of the violations CSV (empty for PASS) |
| `error_message` | Set only for `ERROR` status |
| `meta` | Extra context (column name, expected type, etc.) |
| `run_at` | UTC timestamp of this specific test |

### Violation CSV columns

All original table columns are included, plus:

| Column | Description |
|--------|-------------|
| `_test_name` | Test that flagged this row |
| `_run_at` | UTC timestamp |
| `_test_type` | Category of check |
| `_failed_column` | Which column(s) caused the failure |
| `_expected_type` | (datatype checks only) expected type |
| `_actual_value` | (datatype checks only) raw value that failed |
| `_dup_count` | (no_duplicates only) how many times the key appeared |
| `_duplicate_key` | (no_duplicates only) composite key value |

## Test types reference

### not_null_single
Fails rows where a single column is `NULL`.

```python
{"test_name": "check_id_not_null", "column": "id"}
```

### not_null_composite
Fails rows where **any** of the listed columns is `NULL`. The `_failed_column` value lists which columns were null on each row.

```python
{
    "test_name": "check_order_fields_not_null",
    "columns": ["order_id", "order_date", "status"],
}
```

### no_duplicates
Fails all rows that are part of a duplicate group (based on the key columns). Uses a window function so every duplicate row is captured, not just the first.

```python
{"test_name": "check_order_customer_unique", "columns": ["order_id", "customer_id"]}
```

### datatype_checks
Fails rows where a column's value cannot be cast to the expected type using Athena's `TRY_CAST` / `TRY`. Supported types: `integer`, `decimal`, `timestamp`, `date`, `boolean`, `varchar`.

```python
{"test_name": "check_amount_is_decimal", "column": "amount", "expected_type": "decimal"}
```

### required_rows
Loads a reference CSV from disk and checks that every row (matched on key columns) exists in the Athena table. Missing rows are uploaded as violations.

```python
REQUIRED_ROWS_CONFIG = {
    "enabled": True,
    "csv_path": "config/required_rows.csv",
    "match_columns": ["id", "order_id"],
}
```

## Configuration reference (`config/config.py`)

| Variable | Purpose |
|----------|---------|
| `TABLE_CONFIG` | `database`, `table`, `table_type`, `s3_location` |
| `S3_OUTPUT_CONFIG` | `bucket`, `prefix`, `athena_query_results` |
| `NOT_NULL_SINGLE` | List of single-column null checks |
| `NOT_NULL_COMPOSITE` | List of composite null checks |
| `NO_DUPLICATES` | List of uniqueness checks |
| `DATATYPE_CHECKS` | List of type validation checks |
| `REQUIRED_ROWS_CONFIG` | Reference CSV config |
| `SETTINGS.fail_fast` | Stop after first failure |
| `SETTINGS.max_violations_per_test` | Cap rows captured per test |
| `SETTINGS.log_level` | `DEBUG` / `INFO` / `WARNING` / `ERROR` |

## Changing AWS region

The default region is `ap-south-1`. Override without editing code:

```bash
export AWS_DEFAULT_REGION=us-east-1
python run_tests.py
```

Or edit the `AWS_REGION` line near the top of `run_tests.py`.

## CI / CD integration

`run_tests.py` exits with code `1` if any test fails or errors, making it CI-friendly:

```yaml
# GitHub Actions example
- name: Run DQ tests
  run: python run_tests.py
  env:
    AWS_DEFAULT_REGION: ap-south-1
    ATHENA_WORKGROUP: primary
```
